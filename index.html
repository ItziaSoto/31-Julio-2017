Práctica GIT Configuración Escencial

Recomendación en Commits

  • • Siempre comenzar con letra mayúscula y continuar en minúsculas
  • • No terminar los títulos con un punto
  • • Fuente: http://tomasdel.com/440
Cuando es un commit que requiere de mucha explicación
Escribir el título (no más de 50 caracteres), dar un enter para dejar un espacio en blanco y escribir la descripción del comit
Ejemplo:
Derezz the master control program

MCP turned out to be evil and had become intent on world domination. This commit throws Tron's disc into MCP (causing its deresolution) and turns it back into a chess game.

Nota: Para este commit se debe usar la sentencia git commit para que nos abra el editor de texto y no usar git commit -m "Mi Commit"

Bloque 1

  • echo "# Mario" >> README.md

Bloque 2

git log
Historial de commits
git diff
Muestra los cambios realizados
git commit --amend
Regresa al último commit anterior para editarlo
git checkout --index.html
Regresa el index a un estado anterior (Deshace un cambio o modificación que se haya hecho, antes de hacer el add)
git log --oneline
Historial de commits
git reset head index.html
Cuando se hace un cambio pero ya se ha hecho el add.

Bloque 3 | Ramas

git branch
Muestra todas las ramas.
git branch NombreRama
Crea una nueva rama
git checkout NombreRama
Hace el cambio de rama
git checkout -b NombreRama
Crea una rama y posisciona directamente en ella
git branch -m NombreAntiguo NombreNuevo
Renombre una Rama
git branch -d NombreRama
Elimina una Rama
